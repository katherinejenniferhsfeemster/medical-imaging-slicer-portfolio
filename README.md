## Reproducing everything

All scripts are in `scripts/python/` and use only `SimpleITK`, `numpy`,
`scipy`, `matplotlib`, and `pydicom` — no binary blobs, no large model
checkpoints, no patient data.

```bash
pip install -r requirements.txt
python scripts/python/segmentation_pipeline.py   # Dice / HD95 per organ
python scripts/python/registration.py            # rigid + B-spline, MAE drop
python scripts/python/deidentify_dicom.py        # PS3.15 Basic profile self-test
python scripts/python/render_figures.py          # regenerates every PNG in /docs
```

The 3D Slicer module in `scripts/slicer/SlicerSegmentationReview.py` drops
into Slicer 5.x under *Settings → Modules → Additional module paths*. It
wraps the same pipeline so that a radiologist can click once, get the
baseline segmentation, compute Dice / HD95 against a reference, and export
DICOM-SEG.

## Standards & stack

- **Imaging IO:** DICOM, DICOM-SEG, NIfTI, NRRD, BIDS, RT-STRUCT
- **Analysis:** SimpleITK, ITK, VTK, scipy.ndimage, scikit-image
- **AI:** MONAI, nnU-Net v2, TotalSegmentator, PyTorch
- **Viewers:** 3D Slicer (Python scripted modules, SegmentEditor, Markups, DICOM browser), ITK-SNAP, OHIF
- **Privacy:** DICOM PS3.15 Basic Application Confidentiality profile, deterministic pseudonymisation, HIPAA-aware workflows
- **Ops:** Git, GitHub Actions, Docker, pytest, matplotlib for QC plots
